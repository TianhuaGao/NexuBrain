# Brain Region Connection Index

This index registers independent Obsidian vaults, Git repositories, resource libraries, project folders, and other external knowledge systems outside the generated connection layer.

The goal is to create traceable connections, not to copy external regions into the generated connection layer. Use [[Knowledge/Knowledge-Index|Knowledge]] to map what the external region contains.

## Connection Principles

1. The generated connection layer records direction, tasks, decisions, context, current state, and knowledge maps.
2. External brain regions record their own materials, domain notes, code, experiments, or literature.
3. When the generated connection layer references an external region, prefer this format: region name + relative path or link + note title, file name, commit, or URL.
4. Do not copy large amounts of external-region content into the generated connection layer.
5. If an external region is temporarily unavailable, treat it as a path or environment change, not as evidence of data loss.
