# Pre-Release Checklist

## Content

- [ ] No personal paths, private projects, real credentials, or private data.
- [ ] README explains who the repository serves and what problem it solves.
- [ ] [[Nexus]] navigates to all core areas.
- [ ] [[AGENTS]] is general and not tied to a private environment.
- [ ] Template pages do not contain personal task details.
- [ ] `.nexubrain/templates/RESPONSIBLE_USE.md` is present and defines responsible use boundaries.

## GitHub

- [ ] Choose repository name.
- [ ] Recommended repository name: `NexuBrain`.
- [ ] Choose a license. MIT License is currently recommended.
- [ ] Set repository description.
- [ ] Check default branch name.
- [ ] Before public release, search for sensitive strings such as `token`, `password`, `api_key`, personal paths, phone numbers, and email addresses.

## Optional Improvements

- [ ] Add example vault screenshots.
- [ ] Add an English README summary if publishing a multilingual branch.
- [ ] Add issue templates.
- [ ] Add release notes.
